# 母婴 AI 日历 · 宝塔部署教程

本教程介绍如何将本项目的 **H5 版本** 通过宝塔面板部署到服务器，实现通过域名访问网页版应用。

---

## 一、前置准备

### 1.1 你需要具备

| 项目 | 说明 |
|------|------|
| 云服务器 | 任意 Linux 服务器（如阿里云、腾讯云），建议 1 核 2G 起 |
| 域名（可选） | 用于通过域名访问；无域名也可用 IP + 端口访问 |
| 宝塔面板 | 已安装 [宝塔 Linux 面板](https://www.bt.cn/) 或国际版 aaPanel |
| 本地环境 | 已安装 Node.js（建议 18+），用于在本地执行构建 |

### 1.2 安装宝塔面板（若未安装）

在服务器上执行（根据系统选择）：

```bash
# CentOS
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh

# Ubuntu/Deepin
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh

# Debian
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && bash install.sh
```

安装完成后，记下面板地址、用户名和密码，在浏览器中登录。

---

## 二、本地构建 H5

在**你的电脑**上打开项目目录，执行：

### 2.1 安装依赖

```bash
cd muying-ai-app
npm install
```

### 2.2 执行 H5 构建

```bash
npm run build:h5
```

构建成功后，静态文件会生成在：

```text
dist/build/h5/
```

该目录下的所有文件（如 `index.html`、`assets/` 等）即为需要上传到服务器的内容。

---

## 三、上传到服务器

### 方式 A：宝塔文件管理器上传（适合小项目）

1. 登录宝塔面板 → **文件**。
2. 进入或新建网站根目录，例如：`/www/wwwroot/muying-ai`。
3. 将本地 **`dist/build/h5/`** 目录下的**全部文件和文件夹**上传到该目录（不要只上传 `h5` 文件夹本身，要上传其内部内容，使 `index.html` 在网站根目录）。

### 方式 B：压缩后上传再解压（推荐，速度快）

1. 在本地将 `dist/build/h5` 目录打包为 `h5.zip`。
2. 在宝塔 **文件** 中进入 `/www/wwwroot/muying-ai`（若不存在先新建该目录）。
3. 上传 `h5.zip`，在面板中右键该文件 → **解压**。
4. 若解压后多了一层 `h5` 目录，把 `h5` 里的内容移动到 `/www/wwwroot/muying-ai`，使 `index.html` 位于网站根目录。

### 方式 C：Git 部署（适合经常更新）

1. 在服务器安装 Git（宝塔 **软件商店** 可安装或使用命令行）。
2. 在服务器执行：

```bash
cd /www/wwwroot
git clone <你的仓库地址> muying-ai-app
cd muying-ai-app
npm install
npm run build:h5
```

3. 在宝塔中添加站点时，将**网站根目录**设置为：`/www/wwwroot/muying-ai-app/dist/build/h5`。  
   之后每次更新：`git pull && npm install && npm run build:h5` 即可。

---

## 四、在宝塔中添加网站

### 4.1 添加站点

1. 宝塔面板 → **网站** → **添加站点**。
2. 填写：
   - **域名**：你的域名（如 `h5.xxx.com`），或留空用 IP 访问。
   - **根目录**：`/www/wwwroot/muying-ai`（与你在第三步上传的位置一致）。
   - **FTP、数据库**：按需选择，纯静态 H5 可不创建。
3. 提交后，站点会生成一个默认的 Nginx 配置。

### 4.2 仅用 IP 访问时

- 域名处可填服务器公网 IP，或先随便填一个域名，后面在 **设置** 里改。
- 若需通过 `http://IP:端口` 访问，在 **设置** → **配置文件** 中确认 Nginx 监听对应端口，并在**安全组/防火墙**中放行该端口。

---

## 五、Nginx 配置要点（解决刷新 404、路由与静态资源）

H5 为单页应用（SPA），需要把非根路径的请求都指向 `index.html`，并做好静态资源缓存。

### 5.1 在宝塔中修改配置

1. **网站** → 找到该站点 → **设置** → **配置文件**（或 **伪静态**）。
2. 在 **伪静态** 中填入（推荐）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

若在 **配置文件** 里改，在对应的 `server { ... }` 的 `location /` 块中保留或改为：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

3. 保存后，在 **服务** 中 **重载配置** 或 **重启 Nginx**。

### 5.2 可选：静态资源缓存

在同一个 `server` 里增加（有利于二次访问速度）：

```nginx
location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
    expires 7d;
    add_header Cache-Control "public, immutable";
}
```

保存后重载 Nginx。

## 六、HTTPS（可选但推荐）

1. 网站 **设置** → **SSL**。
2. 选择 **Let's Encrypt**，填写域名并申请（需域名已正确解析到该服务器）。
3. 开启 **强制 HTTPS**，保存。

若使用 IP 或自签名证书，可手动上传证书并同样开启强制 HTTPS。

---

## 七、接口与跨域（若 H5 会请求自己的后端）

- 若 H5 只请求**第三方接口**（如微信、淘宝联盟等），且对方支持跨域，一般无需在宝塔再做配置。
- 若你在**同一台服务器**上部署了后端 API（如 Node、Java 等），建议：
  - 后端单独一个端口或子域名，例如：`api.xxx.com`。
  - 在 H5 的 `src/api/*.ts` 里把请求地址改为该域名。
  - 若后端未设置 CORS，需在后端允许前端域名；或在宝塔该站点的 Nginx 里用 **反向代理** 把 `/api` 代理到后端，从而避免跨域。

示例（API 跑在本机 3000 端口）：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:3000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

| 8 | 若用 HTTPS，证书有效且强制 HTTPS 已开 | □ |

## 九、常见问题

**Q：刷新页面出现 404？**  
A：未配置 SPA 回退到 `index.html`。请按第五节添加 `try_files $uri $uri/ /index.html;` 并重载 Nginx。

**Q：静态资源 404 或白屏？**  
A：确认 `index.html` 与 `assets/` 等在同一级目录，且 Nginx 的 `root` 指向该目录；再检查浏览器控制台报错路径是否与实际上传路径一致（如是否多了/少了目录层级）。

**Q：接口请求跨域？**  
A：优先在后端配置 CORS 允许你的前端域名；或使用 Nginx 反向代理到后端，见第七节。

**Q：更新内容后不生效？**  
A：浏览器强缓存可能导致；可先强刷（Ctrl+F5）。若仍不更新，检查 Nginx 是否对 html 做了长期缓存，或 CDN 缓存是否需刷新。

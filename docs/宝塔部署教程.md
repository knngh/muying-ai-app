# 母婴 AI 日历 · 宝塔部署教程

本教程详细介绍如何将本项目的 **H5 版本** 通过宝塔面板部署到服务器，实现通过域名访问网页版应用。

---

## 一、前置准备

### 1.1 你需要具备

| 项目 | 说明 |
|------|------|
| 云服务器 | 任意 Linux 服务器（如阿里云、腾讯云、华为云），建议 1 核 2G 起 |
| 域名（可选） | 用于通过域名访问；无域名也可用 IP + 端口访问 |
| 宝塔面板 | 已安装 [宝塔 Linux 面板](https://www.bt.cn/) 或国际版 aaPanel |
| 本地环境 | 已安装 Node.js（建议 18+），用于在本地执行构建 |

### 1.2 安装宝塔面板（若未安装）

在服务器上执行（根据系统选择）：

```bash
# CentOS
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh

# Ubuntu/Deepin
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh

# Debian
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && bash install.sh
```

安装完成后，记下面板地址、用户名和密码，在浏览器中登录。

### 1.3 首次登录宝塔配置

1. 首次登录会弹出**"推荐安装套件"**窗口
2. 对于纯静态 H5 网站，选择 **Nginx** 即可（不需要 PHP、MySQL 等）
3. 等待安装完成（约 2-5 分钟）

### 1.4 开放必要端口

在宝塔面板中操作：

1. **安全** → **防火墙** → **添加端口规则**
2. 开放以下端口：
   - `80`（HTTP）
   - `443`（HTTPS）
   - `8888`（宝塔面板，默认）
   - 若用 FTP：`20`/`21`

同时在云服务器控制台的安全组中也要开放对应端口。

## 二、本地构建 H5

在**你的电脑**上打开项目目录，执行：

### 2.1 安装依赖

```bash
cd muying-ai-app
npm install
```

### 2.2 执行 H5 构建

```bash
npm run build:h5
```

构建成功后，静态文件会生成在：

```text
dist/build/h5/
```

该目录下的所有文件（如 `index.html`、`assets/` 等）即为需要上传到服务器的内容。

### 2.3 验证构建产物

构建完成后，检查 `dist/build/h5/` 目录结构：

```text
dist/build/h5/
├── index.html
├── assets/
│   ├── css/
│   ├── js/
│   └── images/
├── static/
└── manifest.json
```

确保 `index.html` 在根目录，不要有额外嵌套层级。

## 三、上传到服务器

### 方式 A：宝塔文件管理器上传（适合小项目）

1. 登录宝塔面板 → **文件**
2. 进入或新建网站根目录，例如：`/www/wwwroot/muying-ai`
3. 将本地 **`dist/build/h5/`** 目录下的**全部文件和文件夹**上传到该目录

> ⚠️ 注意：不要只上传 `h5` 文件夹本身，要上传其内部内容，使 `index.html` 在网站根目录

### 方式 B：压缩后上传再解压（推荐，速度快）

1. 在本地将 `dist/build/h5` 目录打包为 `h5.zip`
2. 在宝塔 **文件** 中进入 `/www/wwwroot/muying-ai`（若不存在先新建该目录）
3. 上传 `h5.zip`，在面板中右键该文件 → **解压**
4. 若解压后多了一层 `h5` 目录，把 `h5` 里的内容移动到 `/www/wwwroot/muying-ai`

### 方式 C：Git 部署（适合经常更新）

1. 在服务器安装 Git（宝塔 **软件商店** 可安装或使用命令行）
2. 在服务器执行：

```bash
cd /www/wwwroot
git clone <你的仓库地址> muying-ai-app
cd muying-ai-app
npm install
npm run build:h5
```

3. 在宝塔中添加站点时，将**网站根目录**设置为：`/www/wwwroot/muying-ai-app/dist/build/h5`
4. 之后每次更新：`git pull && npm install && npm run build:h5` 即可

### 方式 D：宝塔 SSH 终端 + rsync（适合大文件）

在本地终端执行（需先配置服务器 SSH 密钥）：

```bash
rsync -avz --delete dist/build/h5/ root@你的服务器IP:/www/wwwroot/muying-ai/
```

## 四、在宝塔中添加网站

### 4.1 添加站点

1. 宝塔面板 → **网站** → **添加站点**
2. 填写：
   - **域名**：你的域名（如 `h5.xxx.com`），或留空用 IP 访问
   - **根目录**：`/www/wwwroot/muying-ai`（与你在第三步上传的位置一致）
   - **FTP、数据库**：按需选择，纯静态 H5 可不创建
3. 提交后，站点会生成一个默认的 Nginx 配置

### 4.2 仅用 IP 访问时

- 域名处可填服务器公网 IP，或先随便填一个域名，后面在 **设置** 里改
- 若需通过 `http://IP:端口` 访问，在 **设置** → **配置文件** 中确认 Nginx 监听对应端口，并在**安全组/防火墙**中放行该端口

### 4.3 站点目录权限设置

1. 站点添加成功后，点击 **网站** → 找到该站点 → **设置**
2. **目录权限** → 将站点目录权限设置为 `755`，所有者为 `www`
3. 这样可以避免权限问题导致的 403 错误

## 五、Nginx 配置要点

H5 为单页应用（SPA），需要把非根路径的请求都指向 `index.html`，并做好静态资源缓存。

### 5.1 在宝塔中修改配置

1. **网站** → 找到该站点 → **设置** → **配置文件**（或 **伪静态**）
2. 在 **伪静态** 中填入（推荐）：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

若在 **配置文件** 里改，在对应的 `server { ... }` 的 `location /` 块中保留或改为：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

3. 保存后，在 **服务** 中 **重载配置** 或 **重启 Nginx**

### 5.2 完整的 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name h5.xxx.com;  # 替换为你的域名或删除此行用 IP 访问

    # 网站根目录
    root /www/wwwroot/muying-ai;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存（JS/CSS/图片等）
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # HTML 文件不缓存（确保更新立即生效）
    location ~* \.html$ {
        expires -1h;
        add_header Cache-Control "no-store, no-cache, must-revalidate";
    }

    # 安全设置：禁止访问隐藏文件
    location ~ /\. {
        deny all;
        access_log off;
        log_not_found off;
    }

    # 访问日志（可选）
    access_log /www/wwwlogs/muying-ai.log;
    error_log /www/wwwlogs/muying-ai.error.log;
}
```

### 5.3 宝塔可视化配置

如果不想手动编辑配置文件，宝塔也提供可视化配置：

1. **网站** → 站点 **设置** → **伪静态**
2. 选择 **thinkphp** 或 **Vue** 模板（虽然不完全匹配，但能实现 `try_files` 功能）
3. 或者自定义输入上面的规则

## 六、HTTPS 配置（推荐）

### 6.1 免费 SSL 证书（Let's Encrypt）

1. 网站 **设置** → **SSL** → **Let's Encrypt**
2. 选择 **单域名**，填写你的域名
3. 点击 **申请**，等待验证通过
4. 开启 **强制 HTTPS**，保存

> 📌 注意：域名需已正确解析到服务器，否则申请会失败

### 6.2 宝塔自动续期

Let's Encrypt 证书有效期为 90 天，宝塔会自动续期，无需手动操作。

### 6.3 手动上传证书（商业证书）

1. **网站** → **SSL** → **其他证书**
2. 粘贴 `.crt` 证书内容到 **证书(PEM)** 框
3. 粘贴 `.key` 私钥内容到 **密钥(KEY)** 框
4. 开启 **强制 HTTPS**，保存

### 6.4 HTTPS 完整配置示例

```nginx
server {
    listen 80;
    server_name h5.xxx.com;
    # HTTP 自动跳转到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name h5.xxx.com;

    ssl_certificate /etc/letsencrypt/live/h5.xxx.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/h5.xxx.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_session_tickets off;

    # 现代加密配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    root /www/wwwroot/muying-ai;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
}
```

## 七、微信 H5 特殊配置（如需微信授权登录）

如果你的 H5 应用需要在微信中打开并获取微信用户信息，需要额外配置：

### 7.1 微信授权域名

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. **设置与开发** → **公众号设置** → **功能设置**
3. 设置 **网页授权域名** 为你的域名（如 `h5.xxx.com`）

### 7.2 微信 JS 接口域名

1. **设置与开发** → **公众号设置** → **功能设置**
2. 设置 **JS 接口安全域名**

### 7.3 微信内支付（如有）

需要配置 **支付授权目录** 为你的 H5 访问路径

### 7.4 微信 H5 部署注意事项

```nginx
# 解决微信某些机型白屏问题
location / {
    try_files $uri $uri/ /index.html;
    
    # 微信 H5 兼容设置
    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";
}
```

## 八、接口与跨域配置

### 8.1 跨域问题说明

- 若 H5 只请求**第三方接口**（如微信、淘宝联盟等），且对方支持跨域，一般无需在宝塔再做配置
- 若你在**同一台服务器**上部署了后端 API（如 Node、Java 等），需要处理跨域

### 8.2 方案一：后端配置 CORS（推荐）

在后端服务中添加 CORS 头：

```javascript
// Express 示例
app.use((req, res, next) => {
    res.header('Access-Control-Allow-Origin', 'https://h5.xxx.com');
    res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
    res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization');
    next();
});
```

### 8.3 方案二：Nginx 反向代理

若后端未设置 CORS，可在宝塔该站点的 Nginx 里用**反向代理**把 `/api` 代理到后端：

示例（API 跑在本机 3000 端口）：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:3000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    
    # CORS 头（可选，作为备选）
    add_header Access-Control-Allow-Origin $http_origin always;
    add_header Access-Control-Allow-Credentials true always;
}
```

## 九、域名解析

### 9.1 添加解析记录

在域名 DNS 控制台添加：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| A | h5 | 你的服务器公网 IP |
| CNAME | www | 你的域名（如已绑定 IP 可不添加） |

### 9.2 验证解析

```bash
# Windows
ping h5.xxx.com

# Mac/Linux
nslookup h5.xxx.com
```

确认返回的 IP 是你的服务器 IP。

## 十一、性能优化建议

### 11.1 开启 Gzip 压缩

在宝塔 **Nginx 配置** 中添加：

```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_proxied any;
gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml;
gzip_disable "MSIE [1-6]\.";
```

### 11.2 CDN 加速（可选）

1. 在宝塔 **软件商店** 安装 **宝塔 CDN**
2. 或使用阿里云、腾讯云 CDN
3. 将静态资源（JS/CSS/图片）通过 CDN 分发

### 11.3 开启 HTTP/2

在 SSL 配置中确保开启 `http2`：

```nginx
listen 443 ssl http2;
```

## 十四、常见问题

**Q：刷新页面出现 404？**  
A：未配置 SPA 回退到 `index.html`。请按第五节添加 `try_files $uri $uri/ /index.html;` 并重载 Nginx。

**Q：静态资源 404 或白屏？**  
A：确认 `index.html` 与 `assets/` 等在同一级目录，且 Nginx 的 `root` 指向该目录；再检查浏览器控制台报错路径是否与实际上传路径一致（如是否多了/少了目录层级）。

**Q：接口请求跨域？**  
A：优先在后端配置 CORS 允许你的前端域名；或使用 Nginx 反向代理到后端，见第八节。

**Q：更新内容后不生效？**  
A：浏览器强缓存可能导致；可先强刷（Ctrl+F5）。若仍不更新，检查 Nginx 是否对 html 做了长期缓存，或 CDN 缓存是否需刷新。

**Q：HTTPS 证书申请失败？**  
A：检查域名是否已解析到服务器、域名是否正确、服务器防火墙是否放行 80/443 端口。

**Q：访问时出现 "Permission denied" 错误？**  
A：网站目录权限问题。在宝塔中修改目录权限为 755，所有者为 www。

**Q：微信内打开白屏？**  
A：检查是否正确配置了微信授权域名，查看控制台具体错误信息。

## 十五、安全建议

1. **定期更新系统**：保持服务器系统和宝塔面板最新
2. **禁用不必要的端口**：只开放需要的端口
3. **配置防火墙**：使用宝塔防火墙或云安全组
4. **定期备份**：至少每周备份一次网站数据和配置
5. **启用 SSL**：始终使用 HTTPS
6. **监控日志**：定期检查异常访问日志
